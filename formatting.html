Section 1 — Text Formatting

HTML provides many inline elements to annotate text with meaning. Each element carries a specific semantic purpose beyond its visual appearance.

These are all inline elements — they flow within a line of text without breaking the layout.

<strong> — Important

This word is critically important to understand.

<p>This word is
  <strong>critically important</strong>
  to understand.</p>

Semantic importance — not just bold. Screen readers stress the word. Use for genuinely critical content, not decoration.

<em> — Emphasis

I really did not say she stole the money.

<p>I <em>really</em> did not
  say she stole the money.</p>

Stressed emphasis that changes the meaning of a sentence. Screen readers alter intonation. Different from <i> (which is purely stylistic).

<mark> — Highlight

Search results for CSS grid are shown below.

<p>Search results for
  <mark>CSS grid</mark>
  are shown below.</p>

Marks text as relevant to the user's current context — typically search results or a highlighted passage.

<del> & <ins>

Price: CHF 99 CHF 49 — limited offer!

<p>Price: <del>CHF 99</del>
  <ins>CHF 49</ins></p>

<del> marks deleted content (strikethrough); <ins> marks inserted content (underline). Used to show document revisions.

<small>, <sub>, <sup>

Normal text fine print & legal

H2O    E = mc2

<small>fine print</small>
H<sub>2</sub>O
E = mc<sup>2</sup>

<small> for fine print; <sub> for subscript (chemistry, maths); <sup> for superscript (exponents, footnotes).

<code> & <kbd>

Run npm install then press Enter.

The variable let count = 0; is declared here.

<p>Run <code>npm install</code>
  then press <kbd>Enter</kbd>.</p>

<code> for inline code snippets; <kbd> for keyboard input or key names. Both render in a monospace font by default.

<abbr> — Abbreviation

The W3C defines web standards. HTML was created in 1991. 

<abbr title="World Wide Web
  Consortium">W3C</abbr>

Hover over the abbreviations above to see the full form in a tooltip. Improves accessibility for non-expert readers.

<cite> & <q>

As stated in A List Apart:

Good design is obvious. Great design is transparent.

<cite>A List Apart</cite>
<q>Great design is
  transparent.</q>

<cite> references a creative work (book, film, article); <q> marks an inline quotation — the browser adds quotation marks automatically.

Semantic vs presentational: Prefer semantic elements over <b> and <i>, which are purely visual. <strong> and <em> carry meaning that assistive technologies can convey. Style bold/italic text with CSS if it is purely decorative.

Section 2 — Hyperlinks

The <a> element is the foundation of the web. Its href attribute can point to pages, files, email addresses, phone numbers, or anchors within the same page.

Basic link

⌂ Go to Home page 
<a href="index.html">
  Go to Home page
</a>

href can be a relative path, an absolute URL, or any of the special protocols below.

target="_blank" + rel

MDN Web Docs ↗ 
<a href="https://developer.mozilla.org"
   target="_blank"
   rel="noopener noreferrer">
  MDN Web Docs
</a>

target="_blank" opens in a new tab. Always add rel="noopener noreferrer" to prevent the opened page from accessing window.opener — a security best practice.

mailto: & tel:

✉ info@example.com ☎ +41 27 123 45 67 
<a href="mailto:info@example.com">
  info@example.com
</a>

<a href="tel:+41271234567">
  +41 27 123 45 67
</a>

mailto: opens the user's email client; tel: initiates a phone call on mobile devices. Especially useful in responsive design.

Download link

⇓ Download design tokens 
<a href="report.pdf"
   download="annual-report.pdf">
  Download report
</a>

The download attribute forces a download instead of navigation. The attribute value becomes the suggested filename.

Anchor link (same page)

Jump to Quotes section ↓ 
<!-- Link -->
<a href="#demo-quotes">
  Jump to Quotes
</a>

<!-- Target -->
<section id="demo-quotes">
  ...
</section>

The # prefix references a matching id on the page. Used for table of contents, skip navigation, and smooth scrolling.

Button-style link

⌂ Back to Hub 
<a href="index.html"
   class="btn">
  Back to Hub
</a>

/* CSS */
.btn {
  display: inline-block;
  padding: 0.5rem 1rem;
}

Use display: inline-block on an <a> to apply padding and make it look like a button. Keep using <a> for navigation, <button> for actions.

Security note — rel="noopener noreferrer": Without this attribute on target="_blank" links, the newly opened page can access the original page via window.opener and redirect it to a phishing site. Modern browsers partially mitigate this, but always add rel="noopener noreferrer" as a defence-in-depth measure.

Section 3 — Ordered & Unordered Lists

HTML lists group related items. <ul> is for unordered sets; <ol> is for sequences where order matters. Both use <li> for individual items.

<ul> — Unordered list

  • HTML
  • CSS
  • JavaScript
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Bullet points. Order is irrelevant. Change the bullet style with CSS list-style-type.

<ol> — Ordered list

  1. Clone the repository
  2. Install dependencies
  3. Start the dev server
<ol>
  <li>Clone the repo</li>
  <li>Install deps</li>
  <li>Start server</li>
</ol>

Numbered list. Order matters (steps, rankings, instructions). Numbers are generated automatically by the browser.

type & start attributes

type="A"

  1. Alpha
  2. Beta
  3. Gamma

type="i" start="3"

  1. Third
  2. Fourth
  3. Fifth

reversed

  1. Gold
  2. Silver
  3. Bronze
<ol type="A">...</ol>
<ol type="i" start="3">...</ol>
<ol reversed>...</ol>

type: 1 (default), A, a, I, i. start sets the starting number. reversed counts down.

Nested lists

  • Front-end
    • HTML
    • CSS
  • Back-end
    • Node.js
    • Java
<ul>
  <li>Front-end
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Back-end</li>
</ul>

Lists can be nested to any depth. A <ul> or <ol> must always be placed inside a <li>, never directly as a sibling of <li>.

Use lists for navigation too: Most navigation bars are semantically a <ul> of <a> links wrapped in a <nav>. This gives screen reader users the item count and landmark navigation for free.

Section 4 — Definition Lists

The <dl> element groups terms and their definitions. It is ideal for glossaries, metadata, and key-value pairs.

Basic glossary

HTML
HyperText Markup Language — the structure of a web page.
CSS
Cascading Style Sheets — the visual presentation layer.
JavaScript
A scripting language that adds interactivity to web pages.
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup...</dd>

  <dt>CSS</dt>
  <dd>Cascading Style...</dd>
</dl>

<dt> is the term; <dd> is the description. One <dt> can have multiple <dd> elements.

Metadata use case

Author
Jane Smith
Published
September 1, 2025
Category
Web Development
Reading time
8 minutes
<dl>
  <dt>Author</dt>
  <dd>Jane Smith</dd>
  <dt>Published</dt>
  <dd><time datetime="2025-09-01">
    September 1, 2025
  </time></dd>
</dl>

A <dl> styled as a grid makes an elegant metadata panel. The <time> element provides a machine-readable date.

When to use <dl>: Whenever you have pairs of label–value data. Common use cases: glossaries, FAQs (question/answer), API documentation, product specifications, and blog post metadata. Avoid using a table for simple key-value pairs.

Section 5 — Quotes & Citations

HTML provides dedicated elements for quoting external content and attributing sources, both for inline quotations and block-level quotations.

<q> — Inline quote

Tim Berners-Lee once said: The Web is more a social creation than a technical one.

<p>He said:
  <q cite="https://example.com">
    The Web is more a social
    creation than a technical one.
  </q>
</p>

The browser automatically adds quotation marks. The optional cite attribute holds the URL of the source (not displayed to users, but useful for machines).

<blockquote> — Block quote

HTML is the standard markup language for creating web pages. It describes the structure of a web page semantically and originally included cues for the appearance of the document.

— MDN Web Docs
<blockquote cite="https://...">
  <p>HTML is the standard markup
    language...</p>
  <footer>
    — <cite>MDN Web Docs</cite>
  </footer>
</blockquote>

A block-level quotation. Conventionally styled with an indent. The <footer> inside holds the attribution; <cite> names the source.

<cite> — Reference

The concept is explained in CSS: The Definitive Guide by Eric Meyer.

The film The Social Network depicts the founding of Facebook.

<cite>CSS: The Definitive Guide</cite>
<cite>The Social Network</cite>

Used to reference a creative work: book, article, film, song, thesis. Renders in italic by default. Do not use <cite> for a person's name.

Quotes vs citations: <blockquote> and <q> are for the quoted text itself. <cite> is for the title of the source. The cite attribute holds the source URL (machine-readable, invisible to users). These are three distinct concepts — do not confuse them.
