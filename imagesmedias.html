Section 1 — The <img> Element

The <img> element embeds raster images (JPEG, PNG, GIF, WebP, AVIF). It is a void element (no closing tag) and an inline element by default.

Required attributes

<img
  src="mountain.jpg"
  alt="A mountain landscape at sunset"
  width="300"
  height="160">

src is the image path; alt is the text alternative for screen readers and broken images. Always provide a meaningful alt.

Empty alt for decorative images

If an image is purely decorative (adds no information), use an empty alt attribute: alt="".

Screen readers will skip it entirely — which is exactly what you want for background textures, decorative dividers, or icon-only images. 

<!-- Decorative — skip it -->
<img src="divider.png" alt="">

<!-- Informative — describe it -->
<img src="chart.png"
     alt="Bar chart showing 40%
          growth in Q3 2025">

Missing alt is a WCAG failure. Empty alt="" is intentional and correct for decorative images.

loading & decoding

loading="lazy" Defer loading until near viewport — saves bandwidth
loading="eager" Load immediately (default behaviour)
decoding="async" Decode in background — does not block rendering 
<img
  src="hero.jpg"
  alt="Hero banner"
  width="1200"
  height="600"
  loading="lazy"
  decoding="async">

Use loading="lazy" on all images below the fold. Keep loading="eager" for the LCP (Largest Contentful Paint) image — typically the hero.

object-fit explorer (CSS)

img {
  width: 100%;
  height: 200px;
  object-fit: cover;
}

object-fit controls how an image is resized inside a fixed container. Essential for gallery thumbnails and responsive cards.

Always set width and height: Without explicit dimensions, the browser does not know the image's aspect ratio before it loads, causing layout shift (CLS). Always include both attributes matching the intrinsic size of the image. CSS can then override the display size without breaking aspect ratio.

Section 2 — <figure> & <figcaption>

The <figure> element groups self-contained media content (image, diagram, code listing) with an optional <figcaption> label.

Image with caption

<figure>
  <img
    src="diagram.png"
    alt="System architecture diagram"
    width="320"
    height="180">
  <figcaption>
    Fig. 1 — System architecture.
  </figcaption>
</figure>

The <figcaption> can appear before or after the media. Screen readers associate it with the figure automatically.

Code listing as figure

<figure>
  <pre><code>
    function greet(name) { ... }
  </code></pre>
  <figcaption>
    Listing 1 — Greeting function.
  </figcaption>
</figure>

<figure> is not limited to images. It works for any self-contained content: code, charts, videos, audio, tables…

Why not just use <img> + <p>? A <figure> makes the relationship between the media and its caption explicit in the HTML. It can be moved as a unit without losing context, and assistive technologies understand the association.

Section 3 — <picture> & <source>

The <picture> element lets the browser choose the most suitable image from a list of <source> candidates — based on viewport size or supported format.

Modern format with fallback

AVIFBest compression (Chrome, Firefox)
WebPWide support (Safari 14+)
JPEGUniversal fallback for all browsers

The browser picks the first <source> it supports.

<picture>
  <source
    srcset="photo.avif"
    type="image/avif">
  <source
    srcset="photo.webp"
    type="image/webp">
  <img
    src="photo.jpg"
    alt="Mountain view"
    width="800"
    height="500">
</picture>

The <img> inside <picture> is the mandatory fallback and provides the alt text for the whole element.

Art direction (responsive crop)

Serve a different image framing based on viewport width:

Wide viewport: landscape crop
Narrow viewport: portrait crop
<picture>
  <!-- Portrait crop on mobile -->
  <source
    media="(max-width: 600px)"
    srcset="portrait.jpg">
  <!-- Landscape crop on desktop -->
  <img
    src="landscape.jpg"
    alt="Team photo"
    width="1200"
    height="600">
</picture>

The media attribute works like a CSS media query. The browser evaluates sources in order and uses the first match.

<picture> vs srcset: Use srcset on a plain <img> when serving the same image at different resolutions (DPR/size). Use <picture> when you need art direction (different crops) or different formats with fallbacks.

Section 4 — Inline <svg>

SVG (Scalable Vector Graphics) is an XML-based format for two-dimensional graphics. Inline SVG is placed directly in HTML and can be styled and animated with CSS.

Basic shapes

<svg width="200" height="120"
     viewBox="0 0 200 120"
     aria-label="Shapes demo"
     role="img">
  <rect x="10" y="10"
        width="60" height="40"
        fill="#4c7daf" rx="4"/>
  <circle cx="130" cy="35" r="28"
          fill="#66bb6a"/>
  <line x1="10" y1="90"
        x2="190" y2="90"
        stroke="#ef5350"
        stroke-width="3"/>
  <polygon
    points="160,70 185,105 135,105"
    fill="#ff9800"/>
</svg>

The viewBox attribute defines the SVG coordinate system. The width/height set the rendered size on screen.

CSS-controlled SVG

<svg viewBox="0 0 200 100">
  <circle class="icon-circle"
          cx="100" cy="50" r="40"/>
  <path class="icon-check"
        d="M80 50 L95 65 L120 38"
        fill="none"/>
</svg>

/* CSS */
.icon-circle { fill: #4caf50; }
.icon-check  { stroke: white; }

Inline SVG elements can have class attributes and be styled with regular CSS — including hover effects, transitions, and animations.

SVG as icon (accessible)

<button type="button"
        aria-label="Delete item">
  <svg aria-hidden="true"
       focusable="false"
       viewBox="0 0 24 24">
    <path d="..." fill="none"
          stroke="currentColor"/>
  </svg>
  Delete
</button>

Set aria-hidden="true" on decorative icons so screen readers skip them. Use currentColor so the icon inherits its parent's text color automatically.

SVG vs raster images: SVG scales to any resolution without pixelation (perfect for logos, icons, and charts) and is typically smaller in file size than a PNG. Use SVG for anything drawn with shapes and paths. Use raster formats (JPEG, WebP) for photographs.

Section 5 — The <video> Element

The <video> element embeds a native video player. Multiple <source> elements provide format fallbacks across browsers.

Basic video with controls

▶

Video player would appear here with a real .mp4 or .webm file.

<video
  controls
  width="640"
  height="360"
  poster="thumbnail.jpg">
  <source
    src="clip.webm"
    type="video/webm">
  <source
    src="clip.mp4"
    type="video/mp4">
  <p>Your browser does not support
     HTML video.</p>
</video>

controls shows the native browser playback UI. The poster image is displayed before the video plays. The fallback <p> is shown in unsupported browsers.

Key attributes

AttributeEffect
controlsShow browser UI (play, volume…)
autoplayStart playing on load
mutedStart muted (required with autoplay)
loopRestart when finished
posterThumbnail image before play
preloadnone / metadata / auto
 
<video
  autoplay muted loop
  preload="metadata"
  width="800" height="450">
  <source src="bg.webm"
          type="video/webm">
</video>

autoplay requires muted in modern browsers — browsers block autoplay with sound to protect users. Use this pattern for silent background videos only.

Format support: WebM (VP9) offers the best compression for modern browsers. MP4 (H.264) is the universal fallback. Always provide both. Avoid embedding large videos on your own server — use a CDN or a platform like YouTube/Vimeo and embed via <iframe> to save bandwidth.

Section 6 — The <audio> Element

The <audio> element follows the same pattern as <video> but without a visual display area. It supports multiple source formats for cross-browser compatibility.

Basic audio player

♪

Audio player would appear here with a real .mp3 or .ogg file.

<audio controls>
  <source src="sound.ogg"
          type="audio/ogg">
  <source src="sound.mp3"
          type="audio/mpeg">
  <p>Your browser does not
     support HTML audio.</p>
</audio>

Provide OGG first (open format), then MP3 (widest compatibility). The browser uses the first format it supports.

Key attributes

AttributeEffect
controlsShow play/pause/volume UI
autoplayStart on page load (blocked without user gesture on mobile)
loopRestart when finished
mutedStart with audio off
preloadHint to browser: none / metadata / auto
 
<audio controls preload="metadata">
  <source src="podcast.mp3"
          type="audio/mpeg">
</audio>

preload="metadata" loads only duration and format info — a good default that avoids unnecessary bandwidth consumption.

Autoplay and user experience: Most browsers block autoplay on audio unless the user has interacted with the page first. Never autoplay audio with sound — it is considered an accessibility violation and a major UX anti-pattern. Always give users control with the controls attribute.

Section 7 — The <iframe> Element

The <iframe> element embeds another HTML document inside the current page — used for maps, video players, payment forms, and third-party widgets.

Embedding an iframe

<iframe
  src="https://example.com/map"
  title="Interactive map of Sion"
  width="600"
  height="400"
  loading="lazy">
</iframe>

The title attribute is required for accessibility — screen readers announce it as the name of the embedded document.

sandbox attribute (security)

sandboxEnable all restrictions (most secure)
allow-scriptsAllow JS execution inside the iframe
allow-formsAllow form submission
allow-same-originAllow access to same-origin resources
allow-popupsAllow new windows/tabs
<!-- Minimal sandbox -->
<iframe
  src="widget.html"
  sandbox>
</iframe>

<!-- Carefully relaxed -->
<iframe
  src="widget.html"
  sandbox="allow-scripts
           allow-forms">
</iframe>

The sandbox attribute restricts what the embedded document can do. Start with sandbox alone (all restrictions on) and only relax specific capabilities as needed.

Security: clickjacking

Clickjacking attack: An attacker loads your website invisibly in an iframe and overlays a transparent button on top of your site's button — tricking users into clicking something they didn't intend. 
<!-- Server HTTP header -->
X-Frame-Options: DENY

<!-- Modern CSP header -->
Content-Security-Policy:
  frame-ancestors 'none'

To prevent your site from being embedded by attackers, set X-Frame-Options: DENY or the CSP frame-ancestors directive on your server responses.

Performance: Every <iframe> creates a separate browsing context with its own JS engine and document lifecycle. They are expensive. Always add loading="lazy" for iframes below the fold. Prefer a "facade" pattern for embeds (show a thumbnail image; load the iframe only on click).
