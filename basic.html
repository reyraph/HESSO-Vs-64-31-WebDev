Section 1 — Document Structure

Every HTML page is built from the same four mandatory parts: a doctype declaration, a <html> root element, a <head> for metadata, and a <body> for all visible content.

<!DOCTYPE html>

DOCTYPE

Tells the browser to use HTML5 standards mode, not the legacy quirks mode from the 1990s. 

<!DOCTYPE html>

Always the very first line — before anything else, even whitespace. It is not an HTML tag; it is a declaration.

<html lang="en">

The root element wraps the entire document. The lang attribute declares the page language:

  • Helps screen readers use the correct pronunciation
  • Enables browser spell-checking
  • Required for WCAG accessibility compliance
<html lang="en">
  ...
</html>

Use BCP 47 language codes: en, fr, de, fr-CH…

<head>

Contains metadata — information about the page that is not displayed directly:

  • <title> — browser tab title
  • <meta> — charset, viewport, SEO…
  • <link> — stylesheets, favicons
  • <script> — JavaScript files
<head>
  <meta charset="utf-8">
  <title>My Page</title>
</head>

Nothing inside <head> renders visibly on the page.

<body>

Contains all visible content — everything the user sees and interacts with:

  • Text, headings, paragraphs
  • Images, videos, audio
  • Forms, tables, lists
  • Navigation, buttons, links
<body>
  <h1>Hello, World!</h1>
  <p>Visible content here.</p>
</body>

There can only be one <body> per HTML document.

Complete HTML skeleton
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width">
    <title>My Page Title</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>

    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>

    <script src="app.js"></script>
  </body>
</html>
Why the DOCTYPE matters: Without <!DOCTYPE html>, browsers switch into quirks mode and try to render the page like a late-1990s browser, which breaks most modern CSS. Always declare the doctype on line 1.

Section 2 — Headings

HTML provides six heading levels: <h1> to <h6>. They define the document outline and are critical for both accessibility and SEO.

<h1>

Page Title — one per page

<h2>

Major Section

<h3>

Sub-section

<h4>

Sub-sub-section

<h5>

Minor heading

<h6>

Smallest heading

<h1>Page Title</h1>
  <h2>Major Section</h2>
    <h3>Sub-section</h3>
      <h4>Sub-sub-section</h4>
Headings are not about size — they define structure: Never choose a heading level for its visual size. Use CSS for that. Headings define the logical document outline that screen readers and search engines rely on. Only one <h1> per page; do not skip levels (e.g., jumping from <h2> to <h4>).

Section 3 — Paragraphs & Text Flow

<p>, <br>, and <hr> are the core elements for structuring and separating blocks of text.

<p> — Paragraph

This is the first paragraph. It creates a block of text with default top and bottom margins applied by the browser.

This is a second paragraph. Notice the vertical whitespace that separates them.

<p>First paragraph.</p>
<p>Second paragraph.</p>

A block-level element. Each <p> starts on a new line and adds spacing above and below.

<br> — Line Break

Line one of a poem.
Line two continues here.
Line three follows. 

<p>
  Line one of a poem.<br>
  Line two continues here.<br>
  Line three follows.
</p>

A void element (no closing tag). Forces a line break within the same block element. Use sparingly — not as a spacing tool.

<hr> — Thematic Break

Content above the separator.

Content below represents a new topic.

<p>Section one ends here.</p>
<hr>
<p>Section two begins here.</p>

A thematic break that signals a change in topic or subject. Not purely decorative — it carries semantic meaning.

Do not use <br> for spacing: If you need vertical space between elements, use CSS margin or padding. The <br> element is semantically intended for content where line breaks are part of the meaning — such as poetry or postal addresses — not for layout adjustments.

Section 4 — Semantic Elements

HTML5 introduced semantic elements that describe the purpose of their content. They replace generic <div>s with meaningful, self-documenting names. Hover over each zone in the diagram below to learn what it represents.

<header>
<nav>
<main>
<article>
<section>
<aside>
<footer>

<header>

<header>
  <h1>Site Name</h1>
  <p>Tagline</p>
</header>

Site-wide banner at page level, or an introductory block inside an <article>.

<nav>

<nav aria-label="Main">
  <ul>
    <li><a href="/">Home</a></li>
  </ul>
</nav>

Use aria-label when there are multiple <nav> elements on a page to distinguish them.

<main>

<main>
  <h1>Page Title</h1>
  <!-- unique page content -->
</main>

Exactly one <main> per page. Never place it inside <header>, <footer>, or <nav>.

<article>

<article>
  <h2>Blog Post Title</h2>
  <p>Post content...</p>
</article>

Ask yourself: "Could this be shared or distributed independently?" If yes, use <article>.

<section>

<section>
  <h2>Chapter 2</h2>
  <p>Content...</p>
</section>

Groups related content by theme. Should always contain a heading. Not a replacement for a generic <div>.

<aside>

<aside>
  <h3>Related articles</h3>
  <ul>...</ul>
</aside>

Content tangentially related to the main content: sidebars, callout boxes, related links, ads.

<footer>

<footer>
  <p>&copy; 2025 HES-SO Valais</p>
  <nav aria-label="Legal">...</nav>
</footer>

Contains copyright, contact info, secondary navigation. Can also appear inside <article>.

Semantics over presentation: A <div> is a generic container with no meaning. Semantic elements tell browsers, search engines, and assistive technologies what the content represents. This improves accessibility (WCAG) and SEO at zero extra cost. Use <div> only when no semantic element fits.

Section 5 — Block vs Inline

Every HTML element has a default display type: block or inline. Understanding this distinction is essential before learning CSS layout.

Block elements — <div>

Block 1 — takes full width
Block 2 — starts on new line
Block 3 — stacks vertically
<div>Block 1</div>
<div>Block 2</div>
<div>Block 3</div>

Block elements start on a new line and stretch to the full available width. Other block examples: p, h1–h6, section, ul.

Inline elements — <span>

Text with span A and span B flowing inline inside the paragraph. 

<p>
  Text with <span>span A</span>
  and <span>span B</span>
  flowing inline.
</p>

Inline elements flow within text and only take up as much width as their content. Other inline examples: a, strong, em, code.

Key differences

Property Block Inline
Starts on new lineYesNo
Full width by defaultYesNo
Accepts width / heightYesNo
Vertical margin worksYesNo
Can contain block?YesNo
 
/* CSS can override defaults */
span { display: block; }
div  { display: inline; }
div  { display: flex; }

The CSS display property overrides the default. This is exactly how flexbox and grid work.

The golden rule: You can place inline elements inside block elements, but never place block elements inside inline elements. For example, putting a <div> inside a <span> is invalid HTML, and browsers will try to "auto-correct" it in unpredictable ways.
